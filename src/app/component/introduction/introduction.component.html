Introduction



  • The OAM(OPL API Manager) API is organized around RESTful Web Services. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors. We use built-in HTTP features, like HTTP authentication and HTTP verbs (POST), which are understood by off-the-shelf HTTP clients. We support cross-origin resource sharing, allowing you to interact securely with our API from a client-side web application (though you should never expose your secret Id and Client Id API keys in any public website's client-side code).

  • To make the API as explorable as possible, you will have different account for test mode and live mode accecss. Requests made with TEST mode credentials incur no cost but the total number of TEST mode calls will be restricted as per the relevant API. Please contact our business development team for further details.

  • You’ll likely need access to a web developer or programmer (if you’re not one) to get the most use out of the API.


API access will be granted to Customers registered with OPL. To request an access to test APIs please get in touch with OPL Team at support@onlinepsbloans.com with the following details:
